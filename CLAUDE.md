## 📐 작성 규칙

- 각 항목은 **필요할 때만 작성**합니다.
- 모든 항목은 `-` 로 시작하는 **짧고 간결한 문장**을 사용합니다.
- GitHub Release Notes는 **템플릿 전체를 그대로 사용**합니다.
- 일반 커밋 메시지는 템플릿 중 **해당되는 항목만 선택**해 사용해도 됩니다.
- 가능한 한 변경 내용을 **기능 단위로 명확하게 설명**합니다.

---

## 🎯 목적

- 체인지로그를 자동으로 생성할 때 **카테고리별로 정확히 파싱**하기 위함입니다.
- 프로젝트의 모든 변경 내역을 **일관되고 정돈된 형태**로 유지합니다.
- 릴리즈 히스토리 확인 시 **누가 봐도 명확한 변경점 목록**을 남기는 것을 목표로 합니다.
